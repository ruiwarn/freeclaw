# macOS Update and Uninstall Guide

This page documents supported update and uninstall procedures for FreeClaw on macOS (OS X).

Last verified: **February 22, 2026**.

## 1) Check current install method

```bash
which freeclaw
freeclaw --version
```

Typical locations:

- Homebrew: `/opt/homebrew/bin/freeclaw` (Apple Silicon) or `/usr/local/bin/freeclaw` (Intel)
- Cargo/bootstrap/manual: `~/.cargo/bin/freeclaw`

If both exist, your shell `PATH` order decides which one runs.

## 2) Update on macOS

### A) Homebrew install

```bash
brew update
brew upgrade freeclaw
freeclaw --version
```

### B) Clone + bootstrap install

From your local repository checkout:

```bash
git pull --ff-only
./bootstrap.sh --prefer-prebuilt
freeclaw --version
```

If you want source-only update:

```bash
git pull --ff-only
cargo install --path . --force --locked
freeclaw --version
```

### C) Manual prebuilt binary install

Re-run your download/install flow with the latest release asset, then verify:

```bash
freeclaw --version
```

## 3) Uninstall on macOS

### A) Stop and remove background service first

This prevents the daemon from continuing to run after binary removal.

```bash
freeclaw service stop || true
freeclaw service uninstall || true
```

Service artifacts removed by `service uninstall`:

- `~/Library/LaunchAgents/com.freeclaw.daemon.plist`

### B) Remove the binary by install method

Homebrew:

```bash
brew uninstall freeclaw
```

Cargo/bootstrap/manual (`~/.cargo/bin/freeclaw`):

```bash
cargo uninstall freeclaw || true
rm -f ~/.cargo/bin/freeclaw
```

### C) Optional: remove local runtime data

Only run this if you want a full cleanup of config, auth profiles, logs, and workspace state.

```bash
rm -rf ~/.freeclaw
```

## 4) Verify uninstall completed

```bash
command -v freeclaw || echo "freeclaw binary not found"
pgrep -fl freeclaw || echo "No running freeclaw process"
```

If `pgrep` still finds a process, stop it manually and re-check:

```bash
pkill -f freeclaw
```
